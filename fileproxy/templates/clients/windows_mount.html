Windows Mount Client

A lightweight Windows utility that bridges FileProxy's REST API to a native Windows drive letter via WebDAV. No runtime required — single static .exe with no dependencies.

Download
Download 64-bit (.exe) Download 32-bit (.exe)

64-bit recommended for Windows 7 and later. 32-bit for older or 32-bit-only Windows.

  1. 1 Prerequisites

    The Windows WebClient service must be running. Open a command prompt as Administrator and check: 

    sc query WebClient

    If the state is not RUNNING, start it:

    sc start WebClient

  2. 2 Create an API key

    The client authenticates to FileProxy using an API key. To create one:

    1. Click your username in the top-right corner and go to Profile.
    2. Open the API Keys tab and click Create.
    3. Copy the token — it is shown only once.
    Save the token somewhere safe now — you won't be able to view it again.

  3. 3 GUI mode

    Double-click the downloaded .exe to open the GUI. Fill in the fields:

    • Server URL — the base URL of your FileProxy instance (e.g. https://fileproxy.example.com)
    • API Key — the token you created in step 2
    • Drive Letter — the letter to mount as (e.g. F)

    Click Mount. A tray icon appears while the drive is active. Your credentials are automatically saved to %APPDATA%\FileProxyMount\config.json for next time.

  4. 4 CLI mode

    Run from a command prompt to mount without the GUI: 

    fileproxy-mount-amd64.exe mount --server-url https://fileproxy.example.com --api-key YOUR_TOKEN --drive F --port 6789

    Press Ctrl+C to unmount and exit.

  5. 5 Path structure

    Once mounted, each FileProxy connection appears as a top-level folder under the drive letter: 

    F:\connection-name\folder\file.txt
Troubleshooting
Error 67 — "The network name cannot be found"
The WebClient service is not running. Open an Administrator command prompt and run sc start WebClient, then try again.
Error 85 — "The local device name is already in use"
The chosen drive letter is already mapped to something else. Pick a different letter, or release the existing mapping first: 
net use F: /delete /y
Port conflict — "address already in use"
Another process is using the default WebDAV port. Pass a different port with the --port flag: 
fileproxy-mount-amd64.exe mount --port 7890 ...
