Amazon S3 — Setup guide

To connect FileProxy to S3 you'll need two things: an S3 bucket to store your files, and an AWS access key that has permission to use it. This guide walks you through setting both up from scratch — it takes about ten minutes.

  1. 1 Sign in to AWS

    Head to console.aws.amazon.com and sign in. If you don't have an AWS account yet you'll need to create one first.

  2. 2 Create an S3 bucket

    Use the search bar at the top to find S3, then click the orange Create bucket button. A few things to fill in:

    • Bucket name — pick something unique across all of AWS (lowercase letters, numbers, and hyphens, 3–63 characters). Jot it down, you'll paste it into FileProxy later.
    • AWS Region — pick wherever you're based.
    • Leave Block all public access on. It's on by default and should stay that way.

    Scroll to the bottom and hit Create bucket.

  3. 3 Create an IAM user

    Now you need a user whose credentials FileProxy will use. Search for IAM in the top bar, then go to Users → Create user.

    • Give it a name like fileproxy and click Next.
    • Choose Attach policies directly, search for AmazonS3FullAccess, and tick it.
    • Click through to Create user.

  4. 4 Generate an access key

    Click your new user's name in the list, then open the Security credentials tab. Scroll down to Access keys and click Create access key. When AWS asks what you're building, choose Application running outside AWS and click through to finish.

    You'll land on a page showing your Access key ID and Secret access key side by side.

    Copy both values now — AWS won't show the secret access key again after you leave this page.

  5. 5 Paste everything into FileProxy

    Head back to the S3 credentials form and fill in the four fields:

    • Name — a label for your own reference, like prod-backups
    • Bucket name — what you chose in step 2
    • Access key ID — from step 4
    • Secret access key — from step 4

    Hit Save and FileProxy will encrypt your credentials and store them locally. Nothing is sent to any third party.

Want tighter permissions?
AmazonS3FullAccess is the easiest option, but it grants access to all your buckets. If you'd rather limit things to just the one bucket, attach a custom policy instead — paste in the JSON below and swap YOUR-BUCKET-NAME for your actual bucket name: 
{% filter force_escape %}{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["s3:ListBucket"],
      "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME"
    },
    {
      "Effect": "Allow",
      "Action": ["s3:GetObject", "s3:PutObject", "s3:DeleteObject"],
      "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*"
    }
  ]
}{% endfilter %}
