Azure Blob Storage — Setup guide

Connecting FileProxy to Azure requires setting up a service principal — essentially an app identity that FileProxy uses to authenticate. You'll end up with five values to paste in: a storage account name, a container name, and three credentials for the service principal. This guide covers all of it.

  1. 1 Sign in to the Azure Portal

    Go to portal.azure.com and sign in with your Microsoft account.

  2. 2 Create a storage account (skip if you already have one)

    Search for Storage accounts at the top and click + Create.

    • Pick your Subscription and Resource group. If you don't have a resource group yet, the Create new link right there will make one.
    • Storage account name — lowercase letters and numbers only, 3–24 characters. Write it down, you'll need it later.
    • Choose a Region near you and leave the rest as defaults.

    Click Review, then Create, and wait for the deployment to finish.

  3. 3 Create a blob container

    Open your storage account (the Go to resource button on the deployment page is the quickest way). In the left-hand menu, click Storage browser, then click Blob containers, then Add container.

    Give it a name like fileproxy, leave the access level as Private, and hit Create. Note the name down.

  4. 4 Register an app

    Search for App registrations and click + New registration. Name it something recognisable (e.g. fileproxy), leave everything else as defaults, and click Register.

    On the overview page that opens, you'll see two IDs near the top — copy both:

    • Application (client) ID → your Client ID
    • Directory (tenant) ID → your Tenant ID
    Keep this tab open — you'll come back to it in the next step.

  5. 5 Create a client secret

    Still in the app registration, click the Add a certificate or secret link on the overview page, then click + New client secret. Add a description, pick an expiry, and click Add.

    A row appears with your secret. The Value column is what you need — click the copy icon next to it straight away.

    Azure hides the secret value as soon as you navigate away from this page. If you miss it you'll have to delete it and create a new one.

  6. 6 Give the app access to your container

    Now you need to tell Azure that your new app is allowed to read and write blobs. Navigate back to your storage account and click Access Control (IAM) in the left-hand menu.

    Click + Add → Add role assignment. Search for Storage Blob Data Contributor, select it, and click Next.

    Under "Assign access to" leave User, group, or service principal selected, then click + Select members and search for the app name from step 4. Select it, click Select, then Review + assign twice to confirm.

  7. 7 Paste everything into FileProxy

    Head back to the Azure Blob Storage form and fill in the six fields using what you collected above:

    • Name — a label for your own reference, like prod-storage
    • Storage account name — from step 2
    • Container name — from step 3
    • Tenant ID — from step 4
    • Client ID — from step 4
    • Client secret — from step 5

    Hit Save and you're done. FileProxy encrypts the credentials and stores them locally — nothing is sent to any third party.
